## Task
Route a user prompt to the correct Claude model cost tier (S0-S5).

## Tier Definitions
S0 (Haiku low): Trivial -- one-word answers, lookups, yes/no, single shell commands
S1 (Haiku med): Short Q&A, 1-2 sentence explanations, simple factual answers
S2 (Sonnet med): Single-file code edits, 3-10 line patches, targeted fixes
S3 (Sonnet+thinking): Multi-step reasoning, 10-50 lines across 1-2 files, integration work
S4 (Opus+thinking): Architecture design, novel debugging, cross-file refactors, system design
S5 (Opus 1M): Overnight runs, long-context synthesis, full-system audits

## Critical Boundary Rules
- Short prompts can be HIGH tier. "add caching" sounds S0 but requires cache strategy, invalidation, TTL = S3.
- "design" does NOT always mean S4. Designing standard patterns (retry, pagination, logging) = S3.
- Cross-file refactoring and plugin architectures = S4 even if described briefly.
- "fix" or "add" can be S4 if the fix requires understanding system-wide interactions.
- Under-routing is WORSE than over-routing. When uncertain, route UP.
